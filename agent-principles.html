Philosophie de conception

Comment on conçoit les agents CYB

Des principes simples, pensés pour le terrain : un agent qu'on peut utiliser immédiatement, partout, sans dépendre d'une infrastructure client, d'un modèle précis, ou d'une connexion.

# découplé · portable · déterministe · multi-clients

L'esprit

Six principes qui guident chaque agent

Nos missions sont multi-clients, multi-environnements, souvent contraintes (pas de droits admin, données sensibles, réseaux fermés). Pour qu'un agent CYB soit utile partout dès le premier jour, on le conçoit en suivant six règles de bon sens.

Principe 01

Découplé de l'infrastructure client

L'agent doit pouvoir être utilisé immédiatement, sans déploiement ni dépendance à l'environnement d'un client. La forme par défaut est une page HTML autonome : un seul fichier, téléchargeable, qui s'exécute dans n'importe quel navigateur standard.

Pas de serveur à monter, pas d'installation, rien à faire valider par une DSI avant de commencer. On ouvre le fichier, ça marche.

1 fichier HTML Téléchargeable Navigateur standard Zéro install
→Ce que ça change : on démarre une mission en minutes, sur le poste du client comme sur le nôtre, sans dépendre de leur infra.
Principe 02

Découplé du LLM

Aucun agent n'est marié à un modèle. On s'appuie sur une clé API standard pour pouvoir changer de fournisseur à tout moment : modèles Wavestone, accès via Azure, modèles du client, ou LLM locaux.

Le modèle devient un paramètre, pas une fondation. Si le contexte de la mission impose tel ou tel fournisseur, l'agent s'y adapte.

Clé API standard Azure Modèle client LLM local
→Ce que ça change : on respecte la gouvernance de chaque client et on ne subit pas le verrouillage d'un seul fournisseur.
Principe 03

L'usage local toujours possible

Pour les cas les plus complexes ou les plus sensibles, l'agent doit fonctionner en local, sans droits d'administrateur : avec du Python portable et des modèles d'IA locaux.

Réseau fermé, données qui ne doivent pas sortir, poste verrouillé : l'agent reste opérationnel sans rien demander à personne.

Sans admin Python portable Modèles locaux Hors-ligne
→Ce que ça change : aucune donnée sensible ne quitte le poste, même dans les environnements les plus contraints.
Principe 04

Minimiser les appels au LLM

On vise au maximum des systèmes déterministes : la logique qui peut être codée en dur l'est. Le LLM n'est appelé que là où il apporte vraiment de la valeur.

Résultat : un agent qui travaille même sans connexion, de manière fiable et reproductible, sans coût d'API inutile ni dépendance permanente à un modèle.

Déterministe Fiable Reproductible Sobre en tokens
→Ce que ça change : des résultats prévisibles, moins de coûts, et un fonctionnement garanti même hors-ligne.
Principe 05

Multi-utilisateurs, sans usine à gaz

Pas de gestion complexe de droits ni de base de données centralisée. Le partage et la collaboration passent par l'export / import de fichiers JSON.

On garde nos habitudes de travail en équipe, multi-clients et multi-environnements : chacun travaille de son côté, puis on s'échange un simple fichier pour se synchroniser.

Export / import JSON Sans base de données Sans gestion de droits Travail d'équipe
→Ce que ça change : on collabore entre missions et environnements sans monter d'infrastructure partagée.
Principe 06

Cost Management par mission

Quand une mission utilise des agents IA, la proposition commerciale doit mentionner un budget d'usage numérique. Ce budget est associé à une clé API donnée au chef de projet.

Le chef de projet renseigne cette clé dans l'outillage, peut la partager avec son équipe, et suit la consommation comme on suit la consommation de jours sur une mission.

Budget d'usage Clé API mission Chef de projet Suivi conso
→Ce que ça change : l'usage IA devient pilotable, explicite dans la propal, et suivi au même titre que le delivery.
Le fil rouge

Autonomie, portabilité et sobriété. Chaque principe pousse dans la même direction : un agent qu'on peut sortir n'importe où, brancher sur n'importe quel modèle (ou aucun), et faire tourner de manière fiable — sans rien demander à l'infrastructure du client.

C'est ce qui rend nos agents CYB utilisables dès le premier jour d'une mission, et ce qui leur permet de passer à l'échelle quand le besoin grandit.

En une phrase

Un agent CYB, c'est…

…un outil le plus autonome possible, qu'on emporte partout et qu'on partage par fichier.

  • Une page HTML autonome, téléchargeable, qui s'ouvre dans un navigateur standard.
  • Indépendant du LLM : clé API standard, Azure, modèle client ou LLM local.
  • Utilisable en local sans droits admin : Python portable et modèles locaux.
  • Déterministe d'abord : fiable, reproductible, fonctionne hors-ligne.
  • Collaboratif par export / import JSON, sans base ni gestion de droits.
  • Budgété par mission : clé API, enveloppe d'usage et suivi de consommation.
