A

Comprendre la chaîne locale

La logique reste la même que pour une application web : une idée, un repository, un agent IA de code, puis une exécution. La différence est que le résultat tourne sur le PC.

Brique 1

Repository GitHub

Le projet vit dans GitHub pour être historisé, partagé et sauvegardé. GitHub Desktop ou Git SCM synchronise le dossier local.

→
Brique 2

VS Code + Claude

Visual Studio Code ouvre le dossier. L'extension Claude lit le projet, crée les fichiers, corrige les erreurs et propose les commandes.

→
Brique 3

Exécution locale

Un fichier HTML s'ouvre directement dans le navigateur. Une application Python se lance depuis le terminal et peut traiter des fichiers locaux.

Message animateur : l'objectif n'est pas de devenir développeur. L'objectif est de savoir créer un dossier projet propre, demander à l'IA de produire le code, puis tester le résultat sur son poste.

B

Installer les outils sur le PC Wavestone

À faire une fois. Les droits disponibles peuvent varier selon le poste : privilégier les installateurs utilisateur et le Microsoft Store quand c'est possible.

1

Installer Git SCM

Git SCM donne les commandes Git au poste. C'est la couche de synchronisation utilisée par GitHub Desktop, VS Code et les agents de code.

  • Ouvrir git-scm.com/download/win.
  • Télécharger Git for Windows puis lancer l'installation.
  • Garder les options par défaut, notamment Git Credential Manager.
  • Ouvrir un terminal et vérifier avec git --version.
ContrôleTerminal

Une version Git s'affiche. Si Windows demande avec quelle application ouvrir Git, relancer l'installation et garder les options par défaut.

2

Installer GitHub Desktop et se connecter

GitHub Desktop simplifie les premières synchronisations : clone, commit, push et pull sans ligne de commande.

  • Ouvrir desktop.github.com.
  • Installer GitHub Desktop avec l'installateur utilisateur.
  • Se connecter au compte GitHub Wavestone ou au compte autorisé pour l'atelier.
  • Vérifier que l'application affiche le bouton Clone a repository.
3

Installer Python via le Microsoft Store

Python permet de lancer des scripts et des applications locales. Le Microsoft Store évite souvent les problèmes de droits administrateur.

  • Ouvrir le Microsoft Store.
  • Rechercher Python et installer une version récente de Python 3.
  • Ouvrir un terminal et vérifier avec python --version.
  • Vérifier aussi pip --version, qui servira à installer les bibliothèques du projet.
AstuceWindows

Si la commande python ouvre le Store au lieu d'afficher une version, fermer le terminal, le rouvrir, puis réessayer.

4

Installer Visual Studio Code et l'extension Claude

VS Code est l'espace de travail. L'extension Claude permet de demander des changements directement dans le contexte du dossier ouvert.

  • Ouvrir code.visualstudio.com et installer la version User Installer pour Windows.
  • Lancer VS Code puis ouvrir l'onglet Extensions.
  • Rechercher Claude et installer l'extension utilisée par l'équipe.
  • Se connecter au compte autorisé, puis vérifier que le panneau Claude est accessible dans VS Code.
ContrôleVS Code

Le panneau Claude doit pouvoir lire le dossier ouvert et proposer de créer ou modifier des fichiers.

C

Démarrer avec un premier prompt simple

On crée une repository, on la synchronise sur le PC, on l'ouvre dans VS Code, puis on demande à Claude de générer un premier fichier HTML autonome.

But

Valider la chaîne complète

À la fin de cette étape, le fichier index.html est présent dans le dossier local et s'ouvre dans le navigateur. Le système est fonctionnel.

Réflexe

Commiter tôt

Dès que le premier test fonctionne, faire un commit dans GitHub Desktop. Cela crée un point de retour avant les itérations.

1

Créer et synchroniser une repository GitHub

La repository peut être privée si elle contient des éléments internes. Pour un atelier public, rester sur un projet sans données sensibles.

  • Dans GitHub, cliquer sur New repository.
  • Nommer le projet, par exemple local-snake-lab.
  • Cocher Add a README file pour initialiser le projet.
  • Dans GitHub Desktop, cliquer sur Clone a repository, choisir la repository et sélectionner un dossier local.
2

Ouvrir le répertoire avec VS Code

Le dossier cloné devient l'espace de travail. Claude verra les fichiers présents et pourra en créer de nouveaux.

  • Dans GitHub Desktop, cliquer sur Open in Visual Studio Code.
  • Ou dans VS Code, faire File puis Open Folder et sélectionner le dossier du projet.
  • Vérifier que le README apparaît dans l'explorateur de fichiers à gauche.
3

Demander à Claude le jeu Snake en HTML

Le premier prompt doit rester simple et produire un seul fichier. C'est le test le plus rapide pour valider que l'agent peut écrire dans le dossier.

Crée un fichier index.html autonome en HTML/CSS/JavaScript qui propose un jeu de serpent simple.

Contraintes :
- tout doit tenir dans un seul fichier index.html ;
- le jeu doit s'ouvrir directement dans le navigateur depuis mon PC ;
- ajoute un écran d'accueil, un score, un bouton recommencer et des contrôles clavier ;
- garde une interface claire et professionnelle, utilisable en démonstration.
4

Ouvrir le fichier et tester

Une fois le prompt exécuté, le système est utilisable : le code est dans le dossier, Git le voit, et le navigateur lance l'application.

  • Dans VS Code, ouvrir index.html.
  • Faire clic droit puis Reveal in File Explorer, ou ouvrir le dossier depuis Windows.
  • Double-cliquer sur index.html pour l'ouvrir dans Edge ou Chrome.
  • Tester le jeu, puis faire un commit dans GitHub Desktop : Add first local snake app.

À ce stade, la boucle locale est validée : repository synchronisée, édition IA dans VS Code, exécution sur le poste.

D

Cas avancé : transcription audio locale

On passe d'un fichier HTML à une application Python qui télécharge un modèle Whisper depuis Hugging Face, le stocke dans le projet, puis transcrit un audio enregistré au smartphone.

Confidentialité : ne pas déposer d'audio client dans une repository publique. Pour une mission confidentielle, utiliser une repository privée, vérifier le code généré et s'assurer que la transcription ne fait pas d'appel API après téléchargement du modèle.

Principe : le modèle est récupéré une première fois depuis Hugging Face, stocké dans le répertoire du projet, puis réutilisé localement pour les fichiers audio suivants.

1

Créer le prompt applicatif

Le prompt demande une application complète mais garde le périmètre clair : Python, modèle local, dossier d'entrée audio, sortie texte, README et commandes.

Crée une application Python locale pour transcrire des fichiers audio sur mon PC.

Contexte :
- cas d'usage : extraire des informations d'un enregistrement de réunion pour une mission confidentielle ;
- les fichiers audio restent dans le dossier du projet ;
- aucune clé API ne doit être nécessaire pour transcrire ;
- le modèle doit être un modèle Whisper d'OpenAI récupéré depuis Hugging Face, par exemple openai/whisper-small ;
- le modèle doit être stocké localement dans un dossier du projet, par exemple models/whisper-small, pour pouvoir être réutilisé.

Ce que je veux :
- un script Python simple, par exemple transcribe.py ;
- un dossier audio/ pour déposer les fichiers enregistrés avec un smartphone ;
- un dossier transcripts/ pour écrire les transcriptions en .txt ;
- un requirements.txt ;
- un README.md avec les commandes exactes pour installer, télécharger le modèle, puis lancer une transcription ;
- une gestion claire des erreurs si le fichier audio n'existe pas ou si le modèle n'est pas encore téléchargé.

Contraintes :
- privilégie une solution robuste sur Windows ;
- ne pas envoyer le contenu audio vers une API externe ;
- commenter le code uniquement là où c'est utile ;
- prévoir un exemple de commande du type python transcribe.py audio/mon_audio.m4a.
2

Laisser Claude générer le projet

Claude doit créer les fichiers et expliquer les commandes. Relire particulièrement les lignes qui chargent le modèle et le chemin de stockage local.

  • Coller le prompt dans le panneau Claude de VS Code.
  • Accepter la création des fichiers proposés : transcribe.py, requirements.txt, README.md, audio/, transcripts/.
  • Vérifier que le README indique une commande de lancement claire.
  • Faire un commit avant d'installer les dépendances : Add local transcription app scaffold.
3

Installer les dépendances dans un terminal VS Code

Le terminal intégré évite de changer de contexte. Les commandes exactes peuvent varier selon ce que Claude a généré ; le README du projet fait foi.

  • Dans VS Code, ouvrir Terminal puis New Terminal.
  • Créer un environnement local : python -m venv .venv.
  • Activer l'environnement : .venv\Scripts\activate.
  • Installer les dépendances : pip install -r requirements.txt.
  • Lancer la commande de téléchargement ou de premier chargement du modèle indiquée par le README.
AttentionPremier lancement

Le premier téléchargement du modèle peut prendre plusieurs minutes et nécessite une connexion. Les transcriptions suivantes doivent réutiliser le modèle local.

4

Tester avec un audio de smartphone

Le test réel consiste à enregistrer un court audio, le copier dans le projet, puis lancer la transcription depuis le terminal.

  • Enregistrer un audio court avec son smartphone : 30 secondes suffisent.
  • Transférer le fichier sur le PC et le déposer dans audio/.
  • Renommer le fichier simplement, par exemple test-reunion.m4a.
  • Lancer la transcription, par exemple python transcribe.py audio/test-reunion.m4a.
  • Ouvrir le fichier généré dans transcripts/ et vérifier le résultat.

Le système est fonctionnel quand le même projet peut transcrire un nouvel audio simplement en le déposant dans audio/ et en relançant la commande.
