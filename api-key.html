Accès & gouvernance

Créer une clé API pour votre projet

Pour appeler un modèle d'IA depuis votre application, il vous faut une clé API Wavestone. Voici la procédure, la validation requise, et le cas particulier de nos applications maison.

La marche à suivre

Trois étapes pour obtenir une clé

1

Suivre la procédure Power Apps

La création des clés API se fait dans l'application Power Apps Wavestone dédiée. Le product manager du produit peut créer les clés et les associer aux personnes qui en ont besoin.

ouvrir Power Apps ↗
2

Faire valider par le CDM

Toute clé doit être validée par votre Chef de Mission (CDM) avant utilisation : c'est lui qui entérine le besoin et l'usage des appels d'API dans le cadre de la mission.

3

Associer la clé à l'utilisateur

Une fois créée et validée, la clé est associée à la personne (chef de projet / utilisateur) qui réalise la mission, puis utilisée dans le code pour les appels au modèle.

Cas CrisisMaker & CyberBenchExplorer

Si la clé est destinée à utiliser CrisisMaker ou CyberBenchExplorer, ne créez pas de clé vous-même : adressez-vous directement à l'équipe projet, qui vous fournira l'accès adapté à l'application.

Sécurité

Une clé API se traite comme un mot de passe

Règles de sécurité

Une clé API est aussi sensible qu'un mot de passe : quiconque la détient peut consommer le quota et les services associés — à tes frais.

• Ne la partage jamais en clair (email, chat, ticket) et ne la commite pas dans ton code ni sur GitHub.

• Conserve-la dans un coffre-fort de mots de passe (gestionnaire de secrets), pas dans un fichier en clair ni dans tes notes.

• Dans le code, charge-la depuis une variable d'environnement ou un fichier .env exclu du dépôt (via .gitignore).

• En cas de doute sur une fuite, révoque la clé immédiatement et génère-en une nouvelle.
