Le principe fondamental

Tu décris ce que tu veux → l'IA écrit le code → le code est stocké dans un repository → l'application est visible en ligne ou en local. C'est ça, le vibe coding.

Niveau 1 — Web simple

Héberger sur Internet

Le chemin le plus direct : tu codes, tu pousses, c'est en ligne. Idéal pour des pages, des outils et des prototypes.

Chaîne de liaison

Le code va du repository à l'hébergeur, automatiquement
IA → Repo → Build auto → Site en ligne

Connecter des ressources

Ton application a besoin de données, de stockage ou de services tiers ? La chaîne s'allonge.

Chaîne de liaison

IA → Repo → App → BDD / API / Cloudflare / Azure / AWS

REST API Clés API Webhooks
Moyen D

Stack Cloudflare

Un écosystème complet : Workers (serverless), D1 (base SQL), R2 (stockage), KV (clé-valeur), Vectorize (vecteurs), Pages (hébergement).

Workers D1 R2 KV Pages
Gouvernance Wavestone — Hébergement interne

Pour les données clients sensibles, deux options internes sont disponibles :

• Microsoft Azure recommandé — environnement intégré à l'écosystème Wavestone (Azure SQL, Azure OpenAI, Blob Storage, App Service). Privilégié pour rester dans le périmètre maîtrisé.

• Amazon Web Services (AWS) — alternative disponible pour certains cas d'usage (DynamoDB, S3, Lambda, Bedrock). À valider avec la gouvernance IT.

Les clés API Wavestone permettent dans les deux cas de rester dans un périmètre sécurisé et tracé.

Vue d'ensemble

Tous les chemins partent des mêmes 3 briques. La complexité vient de la destination et des ressources connectées.

La règle d'or du vibe coding

1. Stocke
Un repository pour tout historiser
2. Écris avec l'IA
Décris, l'IA code pour toi
3. Exécute
En ligne, en local, ou sur un store

Les niveaux de complexité

Simple

Page web statique, fichier HTML local, script Python. Aucune infrastructure à gérer.

Moyen

Base de données, API tierces, serveur cloud. Nécessite de la configuration et de la sécurité.

Avancé

App Store, application desktop, pipelines CI/CD. Chaîne complète de build et distribution.
