Atelier hands-on

Comment se lancer dans le vibe coding ?

De l’idée au site en ligne, à votre rythme — sans rien installer. Suivez le déroulé, cochez chaque étape, copiez les prompts. Chaque étape est illustrée par une vraie capture d’écran.

# 100% navigateur # sans installation # ~45 min en autonomie # email · github · chatgpt codex

À noter : tout l’atelier se fait avec des comptes grand public et des outils en ligne gratuits, pour se mettre le pied à l’étrier. N’utilisez aucune donnée client ni aucune information interne confidentielle.

Le concept

Trois briques, et c’est tout

Le vibe coding repose sur trois briques qui s’enchaînent. Comprenez-les une fois, vous comprenez tout le reste.

Brique 1

Un agent IA écrit le code

À partir d’une simple idée en langage naturel, un agent IA génère le code source. C’est lui qui fait le travail d’écriture.

Claude · ou ChatGPT (Codex)
Brique 2

GitHub stocke et partage

Le code est déposé dans un espace en ligne qui l’historise et le rend partageable : un dépôt (une « repository »).

GitHub
Brique 3

Le site est publié

Le code est publié sur une page web accessible à tous via une URL. L’hébergement se fait automatiquement.

GitHub Pages

Étape 1 — Préparer

Configurer GitHub & ChatGPT

À faire une seule fois. Comptez 10–15 min.
0/6 étapes
1

Se connecter à GitHub

GitHub stockera votre code. Le plus simple : se connecter avec un compte Google.
  1. Allez sur github.com/login.
  2. Cliquez sur Continue with Google (ou Create an account si besoin).
  3. Choisissez votre compte Google et autorisez.
github.com/login ↗
2

Créer le dépôt (repository) + README

Un dépôt (repository), c’est tout simplement votre projet : un espace en ligne qui regroupe tous ses fichiers et son historique. Le README l’initialise pour qu’il ne soit pas vide.
  1. Cliquez sur Create repository pour créer un dépôt.
  2. Saisissez un nom (ex. cyber-academy), laissez Public, et activez Add README.
  3. Cliquez sur Create repository.
github.com/new ↗
3

Activer GitHub Pages pour mettre votre site en ligne

GitHub Pages publie automatiquement le contenu de la branche main sur une vraie URL. Une branche, c’est comme un répertoire de travail ; main est le répertoire principal de votre projet.
  1. Ouvrez Settings → Pages.
  2. Sous Source, choisissez Deploy from a branch.
  3. Branche main, dossier / (root), puis Save.
  4. Patientez pendant la construction (1–2 min).
  5. Le site est en ligne — mais ne l’ouvrez pas encore (rien à afficher pour l’instant).
pages.github.com ↗
4

Préparer ChatGPT

Codex, l’agent qui écrit le code, vit dans ChatGPT.
  1. Allez sur chatgpt.com.
  2. Connectez-vous (le compte Google fonctionne).
  3. Acceptez les cookies si demandé.
Bon à savoir

Codex est accessible gratuitement, mais la version gratuite est vite limitée : pour l’atelier, il est recommandé de prendre un compte ChatGPT payant (Plus, Team ou Pro). Le principe de configuration est identique avec Claude Code, si vous préférez.

chatgpt.com ↗
5

Ouvrir Codex

Activez l’espace Codex dans ChatGPT.
  1. Dans le menu, ouvrez Codex.
  2. Cliquez sur Learn more.
  3. En haut à droite, cliquez sur Go to Cloud.
  4. Fermez la pop-up de présentation.
Bon à savoir

OpenAI préfère que vous installiez Codex directement sur votre ordinateur : c’est pourquoi de nombreux pop-ups (« Download for Windows », « Install the CLI »…) tentent de vous en dissuader. Ici, on reste volontairement sur la version Cloud, 100 % navigateur.

chatgpt.com/codex/cloud ↗
6

Connecter Codex à GitHub

Autorisez Codex à lire et écrire dans vos dépôts GitHub.
  1. En bas de la page, cliquez sur Connect to GitHub.
  2. La fenêtre « Connect GitHub » s’ouvre : la double authentification (MFA) peut être demandée.
  3. Si besoin, activez la MFA sur votre compte (Turn on MFA).
  4. Continuez vers GitHub.
  5. Autorisez Codex à accéder à GitHub — GitHub s’ouvre.
  6. Revenez sur Codex.
  7. Cliquez sur Configure Repositories on GitHub.
  8. Installez et autorisez l’accès à All repositories.
Bon à savoir

L’activation de la double authentification (MFA) est optionnelle : elle dépend du compte Codex utilisé. Si elle ne vous est pas demandée, passez directement à l’étape suivante.

Étape 2 — Construire

Développer le premier projet

On crée un jeu du serpent qui pose des questions de cybersécurité. Du prompt au site en ligne.
0/5 étapes
1

Décrire le projet à Codex

Nouvelle tâche : choisissez le dépôt, collez le prompt, lancez.
  1. Ouvrez chatgpt.com/codex/cloud.
  2. Sélectionnez votre dépôt de test et la branche main.
  3. Collez le prompt ci-dessous, puis lancez avec le bouton d’envoi (la flèche ↑).
Le prompt du projet
Je suis en charge de notre cyber academy. Pour valider les compétences acquises à la fin des modules, je veux une approche gamifiée. Crée un jeu du serpent dans un fichier index.html autonome (HTML/CSS/JavaScript, sans dépendance externe). À chaque fois que le serpent mange une brique, le jeu se met en pause et pose une question de cybersécurité avec 3 choix : une réponse franchement fausse, une presque juste, et une vraiment juste. Tire au hasard 10 questions parmi une banque de 30, réparties sur l’IAM, la GRC, le SOC et la sécurité de l’IA. Affiche le score de bonnes réponses à la fin : au-delà de 8/10, affiche « Compétences validées ».
Astuce

En demandant explicitement un fichier index.html, vous garantissez que GitHub Pages affichera le projet directement à la racine du site.

2

Laisser Codex coder

Codex travaille seul pendant 3 à 4 minutes. Vous pouvez suivre en direct.
  1. La tâche démarre.
  2. Ouvrez les détails si vous voulez.
  3. Suivez le raisonnement (thinking) de l’agent.
3

Créer puis ouvrir la Pull Request

Au bout de 3 à 4 minutes, le bouton Create PR apparaît en haut à droite : Codex propose alors son code via une Pull Request.
  1. Après 3–4 min, cliquez sur Create PR (en haut à droite).
  2. Après ~30 s, cliquez sur View PR (en haut à droite).
À retenir

Une Pull Request = « je propose ces modifications ». C’est le point de contrôle humain : rien n’est publié tant que vous n’avez pas validé.

4

Valider (merge) la PR dans GitHub

GitHub s’ouvre et montre les changements. C’est vous qui autorisez la publication.
  1. Relisez les changements proposés.
  2. En bas de page, cliquez sur Merge pull request.
  3. Confirmez avec Confirm merge.
  4. La PR est fusionnée et fermée : la publication démarre.
À retenir

Fusionner (merge) la PR sur main, c’est accepter les modifications dans le projet principal : main devient la nouvelle version officielle, et c’est elle que GitHub Pages publie.

5

Vérifier le déploiement et jouer

GitHub Pages reconstruit le site à chaque changement de main. Comptez 1 à 2 minutes.
  1. Revenez dans Settings → Pages.
  2. Vérifiez l’état de mise à jour sous l’URL du site.
  3. Ouvrez l’URL. Si rien ne s’affiche (cache), rechargez ou passez en navigation privée.
  4. Jouez, mangez une brique, répondez aux questions, visez 8/10 !
Bravo

Vous venez de boucler la chaîne complète : idée → prompt → Pull Request → merge → en ligne. Tout le reste n’est qu’une répétition de cette boucle.

Étape 3 — Recommencer

Itérer, encore et encore

Relancez Codex avec un nouveau prompt sur le même dépôt : prompt → Pull Request → merge → en ligne. Quelques idées :

Améliorer le projet
Ajoute des explosions type feu d’artifice à chaque bonne réponse.
Enrichir le contenu
Ajoute 10 questions sur la sécurité des systèmes industriels (OT/ICS).
Nouveau projet
Crée-moi un framework HTML pour mesurer la conformité au document du CSA « Are You Mythos Ready ».
Astuce

Pour un sujet totalement différent, créez un nouveau dépôt et répétez l’Étape 1. Pour faire évoluer un projet existant, restez sur le même dépôt.

Aller plus loin

Il n’y a pas de limite à l’imagination

Une fois la boucle maîtrisée, vous pouvez tout construire. Et l’IA reste votre meilleure alliée à chaque blocage.

Bloqué ? Demandez à l’IA

Toutes les questions sont permises : comment utiliser l’outil, comment ajouter tel élément, comment enrichir le contenu. L’IA est aussi votre tutrice.

Une idée encore floue ?

Demandez à l’IA de vous poser des questions pour préciser votre besoin : « Pose-moi des questions pour m’aider à cadrer mon idée. »

La boucle est toujours la même

Idée → prompt → Pull Request → merge → en ligne. Ce rythme, vous le connaissez désormais. À vous de jouer !

N’oubliez pas l’audit de sécurité !

Avant de partager votre application, demandez à l’IA : « Fais un audit de sécurité de mon application et corrige les vulnérabilités. » Un réflexe simple qui évite bien des erreurs.
