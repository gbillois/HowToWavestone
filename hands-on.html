Atelier hands-on

Comment se lancer dans le vibe coding ?

De l’idée au site en ligne, en direct avec vos participants — sans rien installer. Suivez le déroulé, cochez chaque étape, copiez les prompts.

# 100% navigateur # sans installation # ~45 min en groupe # email · github · chatgpt codex

Astuce animateur : déroulez la page de haut en bas avec le groupe. Cliquez sur les numéros pour cocher les étapes validées, et sur copier pour les prompts. Le sélecteur de langue est en haut à droite.

Le concept

Trois briques, et c’est tout

Le vibe coding repose sur trois briques qui s’enchaînent. Comprenez-les une fois, vous comprenez tout le reste.

Une idée → Du code → Un site en ligne
Brique 1

Un agent IA écrit le code

À partir d’une simple idée en langage naturel, un agent IA génère le code source. C’est lui qui fait le travail d’écriture.

Étape 1 — Préparer

Configurer les environnements

À faire une seule fois avec vos participants. Comptez 10–15 min.
0/5 étapes
1

Vérifier un compte e-mail 5 s

Chaque participant a besoin d’une adresse e-mail pour créer ses comptes. Un compte Gmail personnel est le plus simple, mais n’importe quelle adresse fonctionne.
  1. Ouvrez votre messagerie (ex. gmail.com) dans un onglet.
  2. Si besoin, créez un compte gratuit via Créer un compte.
gmail.com ↗
2

Créer un compte GitHub gratuit

GitHub est l’espace qui stockera le code. Le compte est gratuit.
  1. Allez sur github.com, puis cliquez sur Sign up (en haut à droite).
  2. Saisissez votre e-mail, un mot de passe et un nom d’utilisateur.
  3. Validez le code de vérification reçu par e-mail.
  4. Choisissez l’offre Free.
github.com/signup ↗
3

Créer le dépôt (repository) + README

Un dépôt (repository) est le dossier en ligne qui contient votre projet. Le README l’initialise pour qu’il ne soit pas vide.
  1. En haut à droite, cliquez sur + puis New repository.
  2. Donnez un nom, ex. cyber-academy.
  3. Cochez Public — nécessaire pour la publication gratuite.
  4. Cochez Add a README file.
  5. Cliquez sur Create repository.
github.com/new ↗
4

Activer GitHub Pages

GitHub Pages publie automatiquement le contenu de la branche main sur une vraie URL.
  1. Dans le dépôt, ouvrez l’onglet Settings.
  2. Dans le menu de gauche, cliquez sur Pages.
  3. Sous Source, choisissez Deploy from a branch.
  4. Sélectionnez la branche main et le dossier / (root).
  5. Cliquez sur Save.
  6. Patientez ~1 min : l’URL de votre site s’affiche en haut.
Astuce

Gardez cet onglet ouvert : c’est ici que vous verrez l’URL et que chaque nouvelle version est mise en ligne (https://votre-nom.github.io/cyber-academy/).

pages.github.com ↗
5

Lancer ChatGPT, puis Codex

Codex est l’agent de ChatGPT qui écrit le code directement dans votre dépôt GitHub.
  1. Allez sur chatgpt.com et connectez-vous (le compte Google fonctionne).
  2. Dans le menu de gauche, cliquez sur Codex.
  3. Cliquez sur Connect to GitHub et autorisez l’accès.
  4. Sélectionnez le dépôt cyber-academy.
Bon à savoir

Codex nécessite un forfait ChatGPT payant (Plus, Team ou Pro). Prévoyez-le avant la séance. Claude Code peut jouer le même rôle si vous préférez.

chatgpt.com/codex ↗
Étape 2 — Construire

Développer le premier jeu

On crée un jeu du serpent qui pose des questions de cybersécurité. Du prompt au site en ligne.
0/5 étapes
1

Décrire le jeu à Codex

Dans Codex, créez une nouvelle tâche, vérifiez que le bon dépôt est sélectionné, puis collez le prompt ci-dessous.
  1. Cliquez dans la zone de texte de Codex.
  2. Vérifiez que le dépôt sélectionné est bien cyber-academy.
  3. Collez le prompt, puis cliquez sur Code (ou Envoyer).
Le prompt du jeu
Je suis en charge de notre cyber academy. Pour valider les compétences acquises à la fin des modules, je veux une approche gamifiée. Crée un jeu du serpent dans un fichier index.html autonome (HTML/CSS/JavaScript, sans dépendance externe). À chaque fois que le serpent mange une brique, le jeu se met en pause et pose une question de cybersécurité avec 3 choix : une réponse franchement fausse, une presque juste, et une vraiment juste. Tire au hasard 10 questions parmi une banque de 30, réparties sur l’IAM, la GRC, le SOC et la sécurité de l’IA. Affiche le score de bonnes réponses à la fin : au-delà de 8/10, affiche « Compétences validées ».
Astuce

En demandant explicitement un fichier index.html, vous garantissez que GitHub Pages affichera le jeu directement à la racine du site.

2

Pousser le code (Pull Request)

Quand Codex a fini, il propose une Pull Request : c’est sa façon de proposer d’ajouter son code au dépôt central. Rien n’est publié tant que vous n’avez pas validé.
  1. Relisez le résumé des changements proposé par Codex.
  2. Cliquez sur Create PR (ou Open PR).
  3. Codex ouvre la Pull Request sur GitHub.
À retenir

Une Pull Request = « je propose ces modifications ». C’est le point de contrôle humain : c’est vous qui décidez de publier ou non.

3

Valider la PR dans GitHub

C’est vous qui autorisez la publication, en fusionnant (merge) la Pull Request dans la branche main.
  1. Sur la page de la Pull Request, cliquez sur Merge pull request.
  2. Confirmez avec Confirm merge.
  3. Le code est maintenant dans main : la publication se déclenche automatiquement.
4

Attendre la mise en ligne

GitHub Pages reconstruit le site à chaque changement de main. Comptez 1 à 2 minutes.
  1. Onglet Actions du dépôt : un workflow « pages build and deployment » tourne (rond orange → coche verte).
  2. Ou retournez dans Settings → Pages : le bandeau indique « Your site is live at… ».
5

Voir le résultat !

Ouvrez l’URL de votre site. Le jeu est en ligne, accessible à tous.
  1. Cliquez sur l’URL https://votre-nom.github.io/cyber-academy/.
  2. Jouez, mangez une brique, répondez aux questions, visez 8/10 !
Bravo

Vous venez de boucler la chaîne complète : idée → prompt → Pull Request → merge → en ligne. Tout le reste n’est qu’une répétition de cette boucle.

Étape 3 — Recommencer

Itérer, encore et encore

Relancez Codex avec un nouveau prompt sur le même dépôt : prompt → Pull Request → merge → en ligne. Quelques idées :

Améliorer le jeu
Ajoute des explosions type feu d’artifice à chaque bonne réponse.
Enrichir le contenu
Ajoute 10 questions sur la sécurité des systèmes industriels (OT/ICS).
Nouveau projet
Crée-moi un framework HTML pour mesurer la conformité au document du CSA « Are You Mythos Ready ».
Astuce

Pour un sujet totalement différent, créez un nouveau dépôt et répétez l’Étape 1. Pour faire évoluer un jeu existant, restez sur le même dépôt.

Aller plus loin

Il n’y a pas de limite à l’imagination

Une fois la boucle maîtrisée, vous pouvez tout construire. Et l’IA reste votre meilleure alliée à chaque blocage.

Bloqué ? Demandez à l’IA

Toutes les questions sont permises : comment utiliser l’outil, comment ajouter tel élément, comment enrichir le contenu. L’IA est aussi votre tutrice.

Une idée encore floue ?

Demandez à l’IA de vous poser des questions pour préciser votre besoin : « Pose-moi des questions pour m’aider à cadrer mon idée. »

La boucle est toujours la même

Idée → prompt → Pull Request → merge → en ligne. Ce rythme, vous le connaissez désormais. À vous de jouer !
