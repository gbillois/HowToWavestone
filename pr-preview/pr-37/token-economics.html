Le modèle économique des tokens

Chaque appel à un modèle d'IA consomme des tokens facturés à l'usage. La façon de budgétiser et de suivre cette consommation dépend d'un seul critère : l'usage se fait-il dans le cadre d'une mission client, ou en interne ?

Deux cas, deux circuits

Comment financer et suivre votre consommation de tokens

Le circuit à suivre — budget, suivi, obtention de la clé API — n'est pas le même selon que l'usage est facturé à un client ou réalisé en interne.

Usage client

Usage lié à une mission client

Lorsque l'IA est utilisée dans le cadre d'une mission facturée, le coût des tokens est porté par le client et doit être anticipé dès le chiffrage.

  1. 1Budgétiser dans la proposition commerciale — provisionner une ligne « usage IA / tokens » dès le chiffrage de la propal, dimensionnée selon les volumes d'appels prévus.
  2. 2Suivre la consommation dans Wavekeeper — une fois la mission démarrée, la consommation réelle de tokens est suivie dans Wavekeeper, au même titre que le reste du budget de la mission.
  3. 3Obtenir la clé auprès du product owner de l'outil — la clé API donnant accès à l'outil utilisé dans la propal (CrisisMaker, CyberBenchmark Helper…) est fournie par le product owner de cet outil, qui la rattache à la mission.
Usage interne

Usage interne, hors mission

Pour les agents utilisés en interne — R&D, outillage, expérimentation — il n'y a pas de propal, donc pas de budget client dédié. Le financement se fait sur budget interne, après validation.

  1. 1Faire une demande au CDM — toute consommation de tokens hors mission doit être validée par un Chef De Mission (CDM), qui autorise le besoin et son financement sur budget interne.
  2. 2Créer la clé API — une fois la demande validée, la clé se crée selon la procédure standard.
voir la procédure de clé API →

Un usage de tokens non budgété en propal fait courir un risque de dépassement à la mission : anticipez toujours le volume prévu avant de chiffrer, et faites confirmer ce budget par le product owner de l'outil concerné.

Critère Usage client (mission) Usage interne (hors mission)
Budget Provisionné dans la proposition commerciale Budget interne, après validation du CDM
Suivi Consommation suivie dans Wavekeeper Suivi interne piloté par le CDM
Clé API Délivrée par le product owner de l'outil utilisé dans la propal Créée après validation, via la procédure clé API
