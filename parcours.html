Parcours 01

Application sur ma machine

Git + VS Code + Python, avec sauvegarde et travail en commun via GitHub. Tout reste en local sur ta machine.
0/6 étapes
1

Tester Claude + créer un GitHub

Connecte-toi à Claude (compte Wavestone) et crée un compte GitHub gratuit pour stocker et versionner ton code.
claude.ai/new ↗github.com ↗
2

Installer Git

Télécharge et installe Git (version Windows). Pas d'interface : c'est le composant qui suit les modifications de tes fichiers.
git-scm.com/install ↗
3

Installer GitHub Desktop

Le client qui synchronise tes repositories entre le web et ta machine (Documents/GitHub). Connecte ton compte GitHub. C'est lui qui permet la sauvegarde et le travail en commun.
desktop.github.com ↗
4

Installer Visual Studio Code

L'éditeur dans lequel tu travailleras. Télécharge pour Windows, puis Suivant → Installer.
code.visualstudio.com ↗
5

Installer Python pour le code avancé

Installe le Python Install Manager (recommandé via le Microsoft Store). Utile dès que Claude doit générer et exécuter du Python en local.
Microsoft Store ↗python.org ↗
6

Extension Claude Code + premier fichier

Dans VS Code : Extensions (Ctrl+Shift+X) → cherche « Claude Code » (Anthropic) → Installer. Connecte ton compte Claude, ouvre un dossier de ta machine, puis demande ton premier fichier dans le panneau Claude (icône +).
guide officiel ↗ouvrir dans VS Code ↗
Exemple — page simple
Crée-moi une page HTML autonome nommée tetris.html : un Tetris minimaliste qui démarre sur la touche Entrée.
Exemple — script Python local
Crée-moi un script Python qui utilise en local faster-whisper pour transcrire mes fichiers audio et vidéo sur ma machine.
Human in the loop

En local, Claude demande ta permission avant d'exécuter une commande. Lis ce qu'il propose, puis valide — rien ne se lance dans ton dos.

Parcours 03

Appeler un modèle d'IA dans mon appli

Intégrer des appels à un modèle d'IA via des clés API, dans l'environnement Wavestone.
0/3 étapes
Règle de gouvernance

Les données clients sensibles se manipulent dans l'environnement Microsoft. Pour les traitements, privilégie donc les appels d'API via Microsoft plutôt qu'un traitement local non maîtrisé.

Tes use cases reposant sur des outils web autonomes qui font des appels API peuvent ainsi passer à l'échelle.

1

Créer une clé API Wavestone point de départ

Le product manager du produit peut créer des clés API et les associer aux personnes qui réalisent les missions. Rends-toi sur la page dédiée (voir « Infos & liens utiles »).
aller aux liens ↓
2

Associer la clé à l'utilisateur

Depuis l'interface, associe la clé créée à la personne (chef de projet / utilisateur) qui en a besoin.
3

Appeler le modèle depuis ton code

Utilise la clé dans ton application pour faire les appels API vers le modèle, en restant dans l'environnement Wavestone / Microsoft pour les données sensibles.
